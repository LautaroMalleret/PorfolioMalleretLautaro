API - CANCHASBA

Proyecto enfocado en el diseño y ejecucion de una API-REST.

Tecnologías utilizadas: Java, Springboot, Git, SQL, Maven

Desarrollé una API REST para un portal de gestión de canchas de fútbol en la provincia de Buenos Aires. La API permite consultar, agregar, modificar y eliminar información sobre canchas, incluyendo detalles como el nombre, dirección, zona, tipo de cancha, cantidad de canchas, teléfono de contacto y calificación de usuarios. Implementé endpoints para filtrar canchas por diversos parámetros, como nombre, zona y tamaño. Además, la API gestiona la autenticación y las variables de configuración a través de entorno, y está diseñada para ser fácilmente consumida por aplicaciones web y móviles.

Backend : JAVA 17 con SPRINGBOOT
Base de datos : POSTGRESQL (manejado con JPA y gestor DBEAVER)
Control de versiones : GIT
Manejador de dependencias : MAVEN
Despliegue : RENDER con DOCKER
Documentacion : SWAGGER

Clusters - graficador de clusters en mapas

aplicacion de algoritmos de grafos, específicamente el algoritmo de Kruskal, para dividir puntos en grupos basados en su proximidad, creando un árbol generador mínimo y clusters. Esto permitió resolver problemas complejos de análisis espacial

Tecnologías utilizadas: Java, UML, Git, JMapViewer

cursando la materia de programación 3, realice el siguiente proyecto que consistió en crear un software de escritorio en Java que permitía al usuario ingresar puntos y luego graficarlos en un mapa separandolos en grupos dependiendo la distancia entre los mismos. Se usaba un algoritmo de grafos de nodos para crear un árbol generador mínimo con el algoritmo de Kruskal, para luego dividirlo y crear clusters de puntos cercanos. El proyecto se centró en trabajar con grafos de nodos y en la aplicación de algoritmos para resolver problemas complejos. Dicho proyecto me permitió aprender a cómo aplicar algoritmos para resolver problemas de manera efectiva en la programación.

Wordle ARM - Assembler

Enfoque en desarrollo en assembler ARM a bajo nivel con manejo de memoria, archivos, implementación de subrutinas e interrupciones.

Tecnologías utilizadas: Assembler ARM

Este proyecto lo diseñé y fue utilizado como trabajo practico integrador por todas las comisiones de la materia "Organizacion del computador" de la Universidad Nacional de General Sarmiento. Este, consiste en desarrollar el juego Wordle utilizando ARM en ensamblador, el cual debía ejecutarse en terminal a través de una conexión remota a una Raspberry Pi. A través de este proyecto, los alumnos pusieron en práctica conceptos de arquitectura ARM, como el manejo de memoria, uso de subrutinas e interrupciones, así como la manipulación de archivos y colores en terminal. La evaluación consideraba estos aspectos junto con el funcionamiento completo del juego, incluyendo un informe detallado con pseudocódigo y dificultades encontradas.

Dashboard

Desarrollo de una herramienta interna pensada para facilitar la gestión de productos por parte del dueño del catálogo, priorizando simplicidad de uso, bajo costo y total autonomía en la administración del contenido.

Tecnologías utilizadas: React, TypeScript, Node.js, MongoDB Atlas.

Este panel administrativo fue desarrollado como parte del sistema completo de catálogo online de indumentaria. Permite al administrador gestionar los productos del catálogo en tiempo real: Agregar, editar y eliminar productos. Subir imágenes y asociarlas a cada producto. Filtrar productos por tipo, prenda o nombre. Controlar stock, precio, talles y variantes. El acceso está protegido mediante login con autenticación por cuenta de Google. La experiencia está pensada para ser clara y directa, sin necesidad de conocimientos técnicos para su uso.
Frontend: React + Vite con TypeScript
Estilos: Tailwind CSS
Autenticación: JWT
Gestión de imágenes: ImageBB API
Backend: Node.js con Express
Base de datos: MongoDB Atlas (con Mongoose)
Control de versiones: Git + GitHub
Despliegue: Render
Uptime: UptimeRobot (ping periódico para mantener desplegada la api)
